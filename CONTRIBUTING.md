# Contributing

First off, thanks for taking the time to contribute!

## Getting Started

1. Fork the repository and create a branch from `main`.
2. Make your changes.
3. Open a pull request — a maintainer will review it.

## Reporting Issues

Use the GitHub issue tracker. Please include:
- A clear description of the problem
- Steps to reproduce
- The repository and workflow configuration you're using (redact any secrets)

## Pull Requests

- Keep PRs focused — one logical change per PR.
- Update documentation (`README.md`) if your change affects behaviour or inputs.

## Code of Conduct

This project follows the [Datadog Code of Conduct](https://github.com/DataDog/.github/blob/main/CODE_OF_CONDUCT.md).
